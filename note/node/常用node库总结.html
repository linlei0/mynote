# 常用 node 库总结

  • fs-extra (opens new window)

    对 fs 的扩展

  • user-home (opens new window)

    获取当前用户主目录

  • ejs (opens new window)

    模版渲染

  • glob (opens new window)

    遍历文件

  • kebab-case (opens new window)

    驼峰转"-"连接

    例如：WebkitTransform -> -webkit-

  • semver (opens new window)

    版本比对工具

  • colors (opens new window)

    终端文本颜色设置

  • chalk (opens new window)

    终端文本颜色设置

  • commander (opens new window)

    开发脚手架常用命令行工具

  • inquirer (opens new window)

    常见的交互式命令行用户界面和 commander 组合开发 cli 的利器

  • lerna (opens new window)

    开发脚手架常用框架

  • dotenv (opens new window)

    读取.env 文件库，用于处理一些常用的配置文件

  • import-local (opens new window)

    用于判断是否使用本地版本和开发版本

const importLocal = require("import-local");

function logInfo(info) {
  require("npmlog").info("cli", info); // 开发版本
}
// console.log(__filename)
// console.log(__dirname)
if (importLocal(__filename)) {
  // require('npmlog').info('cli', '正在使用本地版本！') // 发布npm的包下载到本地的package
  logInfo("正在使用本地版本！");
} else {
  // require('npmlog').info('cli', '正在使用开发版本') // 开发版本
  logInfo("正在使用开发版本");
  require("../lib")(process.argv.slice(2));
}

  • npmlog (opens new window)

    终端打印日志工具

  • path-exists (opens new window)

    判断文件目录地址是否存在

  • root-check (opens new window)

    对 root 权限进行降级

const rootCheck = require("root-check");
rootCheck();
console.log(process.getuid());

    用于 git 在任何 node.js 应用程序中运行命令的轻量级界面

  • terminal-link (opens new window)

    在终端种进行 url 浏览器地址跳转

  • npminstall (opens new window)

    npm install 更快速和方便

  • pkg-dir (opens new window)

    查找 Node.js 项目或 npm 包的根目录

  • url-join (opens new window)

    拼接 url 地址

  • cli-spinner (opens new window)

    spinner 加载器
